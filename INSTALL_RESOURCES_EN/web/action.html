Collect
Collecting data consist in identifying their source and producer. In the platform, it means creating and saving a new dataset, with at least a few metadata like the title and a short description. For remote source datasets, it can be the source setup (with remote URL), setting up a scheduler and verifying that the connectivity works properly.
Edit
Editing a dataset consist in processing the data according to best practices defined within the organization. The scope of this step can be very large, but mainly its goal is to process the data to make it compatible with standards, aligned with business requirements easier to reuse by enriching the data with additional information like labels or geographical coordinates. Technically, it's also appropriate to check if the schema is correct, fields formats and types are correctly defined regarding the content of each columns. Finally, it can be interesting to detail each fields with a description to ease the understanding of the data.
Format
Format a dataset consist in defining its default views and polish the presenting. Like selecting, in first, most important fields for the table view, setting the correct title for the map tooltip, reducing the list of visible fields to help users to go straight to the essential content. This part is often a cooperative work between stakeholders and business users or representatives whom can tests the readability and if the data is intelligible.
Update
Sometimes complex due to schema modification for example, a dataset in an update production cycle means that the dataset is stuck waiting for a data publisher or administrator to operate on it. Sometimes for security or administrative reasons, or data modifications that force to reevaluate the needs and purposes of the published data, it's important to distinguish datasets in this state.
Finalized
A finalized dataset is validated, and ready to be (or already) published. All necessary operations has been performed and the dataset is ready. (Defining metadata, processing the data, formatting the overall dataset, scheduling the source and more...)
Unsupervised
Some datasets don't need to be supervised or even updated. They are unsupervised as they contain static data, or simply data that are not concerned by any supervision in their organization. For examples, a geo referential federated from Opendatasoft data-network, or a data covering a limited period of time like a festival program on a specific date.
